# WinSpeedBall 隐私说明

生效日期：2026-07-11

## 1. 适用范围

本说明适用于 WinSpeedBall 浏览器扩展对用户数据的处理。当前版本没有项目自建账户服务器，注册与登录均为当前浏览器内的本地账户功能。

## 2. 本地保存的数据

- 本地账户：用户名、显示名称、随机盐、密码摘要、账户创建与更新时间。
- 临时会话：当前登录账户标识、签发时间和失效时间；浏览器会话结束后失效。
- 功能数据：扩展设置、OCR 结果、AI 历史、任务日志、截图和用户导入的脚本。
- 开发者数据：SDK 草稿、能力授权记录和按脚本隔离的键值数据。
- 临时 SDK 数据：运行令牌、运行会话和一次性页面上下文确认；只保存在浏览器会话存储中并设有短期有效期。
- 使用声明记录：声明版本、内容 SHA-256 摘要、确认时间、产品版本和本地用户标识。

密码不以明文保存。账户密码通过 PBKDF2-SHA-256 和随机盐生成摘要。

## 3. 数据发送

- Tesseract OCR 默认在本地运行，截图不会因 OCR 自动上传到项目服务器。
- 用户主动发送或开启自动发送后，OCR 或网页文字会发送到用户选择并配置的 AI 服务。
- 用户导入的脚本可能按其权限声明访问网页或网络。用户应只导入可信脚本。
- SDK 脚本只能调用用户确认过的 `WSB` 能力。脚本声明并调用 `ai.request` 时，输入文本会发送到用户选择的 AI 服务；SDK 不向脚本公开 API Key。
- 当前版本不会把本地账户资料发送到 WinSpeedBall 项目服务器。

第三方 AI 服务按照其自身条款和隐私规则处理数据，用户应在发送前确认内容适合交由该服务处理。

## 4. 数据用途

数据仅用于提供用户主动使用的扩展功能、恢复本地设置、显示历史和维护本地登录状态，不用于基于浏览活动的个性化广告。

## 5. 保存与删除

- 用户可以在账户页删除本地账户。
- 删除账户不会自动删除截图、OCR、AI 历史、日志和设置，避免误删其他本地功能数据。
- 用户可以通过浏览器的扩展数据清除功能删除全部本地数据。
- 扩展内的“隐私中心”可以分别查看并删除截图、OCR、AI 历史、日志、用户脚本和账户数据。删除脚本数据会同时撤销 SDK 授权、令牌和会话，并清理 SDK 草稿与隔离存储。
- 在隐私中心删除“账户数据”会同时清除登录会话、活动 Provider 选择以及本机声明确认与历史；再次使用时需要重新确认声明。
- 卸载扩展通常会由浏览器删除扩展存储，具体行为以浏览器实现为准。

## 6. 安全边界

扩展采取最小权限、受信任存储访问、加盐密码摘要、短期会话、能力确认和独立沙箱 Worker 等措施。浏览器扩展本地存储并非专用加密保险库；可以控制本机浏览器配置或操作系统账户的人仍可能访问或修改本地数据。旧脚本兼容层拥有比新 SDK 更宽的能力，只应运行来源可信且已核对权限声明的脚本。

## 7. 变更与联系

数据处理方式发生重要变化时，扩展会更新声明版本并要求用户重新确认。问题或删除协助可通过 [GitHub Issues](https://github.com/wodengnisinian/WinSpeedBall-Extension/issues) 联系项目维护者。
